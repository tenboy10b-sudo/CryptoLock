Підключив зовнішній диск — Windows його не бачить. Найчастіше проблема вирішується за 5 хвилин без сторонніх програм.

---

## Крок 1: Перевір фізичне підключення

- Спробуй інший USB порт (краще напряму на материнській платі, а не через хаб)
- На зовнішньому HDD — перевір чи підключений кабель живлення якщо є
- Спробуй інший USB кабель — вони часто виходять з ладу
- Підключи до іншого ПК — щоб зрозуміти чи проблема в диску чи в ПК

---

## Крок 2: Перевір Диспетчер пристроїв

`Win + X` → **Диспетчер пристроїв** → розгорни **Дискові пристрої** і **Контролери USB**.

- Жовтий трикутник — проблема з драйвером
- Пристрій є але без букви — перейди до кроку 3
- Пристрою немає взагалі — проблема фізична або з живленням

```powershell
# Оновити драйвер USB контролера
Get-PnpDevice | Where-Object { $_.Class -eq "USB" -and $_.Status -ne "OK" } |
  Update-PnpDevice -Confirm:$false
```

---

## Крок 3: Призначити букву диска через Керування дисками

Найчастіша причина — диск визначається але без букви.

`Win + X` → **Керування дисками** (або `Win + R` → `diskmgmt.msc`)

Знайди диск у нижній частині вікна. Якщо він є але без букви:

Правою кнопкою на розділ → **Змінити букву диска або шлях до диска** → **Додати** → вибери букву → **OK**.

Якщо розділ відображається як **Нерозподілений** — диск потрібно ініціалізувати або форматувати (дані будуть втрачені).

---

## Крок 4: Ініціалізація нового диска

Новий диск що ніколи не використовувався потребує ініціалізації.

В **Керуванні дисками** → правою кнопкою на диску (не на розділі) → **Ініціалізувати диск** → вибери **GPT** (для дисків > 2 ТБ або UEFI систем) або **MBR** → OK.

Потім: правою на нерозподілений простір → **Створити простий том** → пройди майстер.

---

## Крок 5: Перевірити через diskpart

```cmd
diskpart
list disk
```

Якщо диск є в списку — він фізично визначається. Проблема в розділах або букві.

```cmd
select disk 1
list partition
detail disk
```

---

## Крок 6: Виправити помилки на диску

```cmd
rem Замінити X на букву диска
chkdsk X: /f /r
```

Детальніше: [Перевірка диска chkdsk](/perevirka-dysku-na-pomylky-chkdsk)

## Типові проблеми

### Диск показує 0 байт або "Носій не вставлено"

Файлова система пошкоджена або диск відформатований під іншу ОС (ext4 для Linux, APFS для macOS).

- Linux ext4 → Windows не читає нативно. Потрібен Ext2Fsd або WSL
- macOS APFS → потрібен HFSExplorer або Paragon

### Диск клацає при підключенні

Фізична несправність — механічний збій. Негайно:
1. Вимкни живлення
2. Не намагайся "постукати" або розкрити самостійно
3. Зверніться до сервісу відновлення даних

### Повідомлення "Вам потрібно відформатувати диск"

НЕ форматуй одразу — дані можна врятувати. Спробуй:

```cmd
chkdsk X: /f
```

Або програму відновлення: [Як відновити видалені файли](/yak-vidnovyty-vydaleni-fayly-windows)

## Підсумок

Порядок перевірки: інший USB порт → Диспетчер пристроїв → Керування дисками (призначити букву) → diskpart → chkdsk. У 90% випадків проблема вирішується на кроці 3 — диск є але без букви.
