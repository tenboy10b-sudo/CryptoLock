Якщо Windows завантажується більше 30–60 секунд — щось не так. На SSD має бути 10–15 секунд, на HDD 45–90. Ось як знайти причину і виправити.

---

## Крок 1: Виміряй час завантаження

Перш ніж щось міняти — зафіксуй поточний час.

```powershell
# Час останнього завантаження
(Get-Date) - (gcim Win32_OperatingSystem).LastBootUpTime

# Детальний звіт часу завантаження
Get-WinEvent -ProviderName "Microsoft-Windows-Diagnostics-Performance" |
  Where-Object { $_.Id -eq 100 } |
  Select-Object TimeCreated,
    @{N="Boot Time (s)"; E={$_.Properties[0].Value / 1000}} |
  Select-Object -First 5
```

Або через **Диспетчер завдань** → вкладка **Автозапуск** → внизу показано **Час до останнього завантаження BIOS**.

---

## Крок 2: Вимкни зайве з автозапуску

Найбільший вплив на час завантаження.

`Ctrl + Shift + Esc` → **Автозапуск** → сортуй за **Вплив на запуск**

Вимкни все з позначкою **Високий** яке не потрібно при старті:
- Steam, Discord, Spotify — запускай вручну коли потрібно
- OneDrive — можна вимкнути якщо не синхронізуєш постійно
- Dropbox, Google Drive — теж за потребою
- Принтерне ПЗ — зазвичай не потрібне при старті

**Залиш:** антивірус, аудіо-драйвери (Realtek), відеодрайвери.

Детальніше: [Як вимкнути автозавантаження](/yak-vymknuty-avtozavantazhennya)

---

## Крок 3: Увімкни швидкий запуск

Швидкий запуск (Fast Startup) зберігає стан ядра при вимкненні і прискорює наступний старт.

`Пуск` → `Параметри` → `Система` → `Живлення та сон` → **Додаткові параметри живлення** → **Вибрати дії кнопок живлення** → **Увімкнути швидкий запуск**

> Якщо використовуєш подвійне завантаження (Windows + Linux) — краще вимкни, може спричинити проблеми з доступом до диску.

---

## Крок 4: Перевір HDD vs SSD

На HDD Windows завантажується в 3–6 разів повільніше ніж на SSD. Це найрадикальніше покращення.

Як перевірити що стоїть:

`Ctrl + Shift + Esc` → **Продуктивність** → **Диск** → якщо тип "HDD" — пора міняти.

Перехід описано: [Як перенести Windows на SSD](/yak-perenestyy-windows-na-novyy-dysk-ssd)

---

## Крок 5: Оптимізуй службу Windows Update

Windows Update може запускати сканування одразу після завантаження і гальмувати систему 5–15 хвилин.

```powershell
# Перевірити чи Update завантажує щось у фоні
Get-Service -Name wuauserv | Select-Object Status

# Обмежити пропускну здатність оновлень
# Параметри → Windows Update → Додаткові параметри → Оптимізація доставки
# → Додаткові параметри → обмеж до 20% у фоні
```

---

## Крок 6: Перевір завдання в планувальнику

Деякі програми реєструють завдання які виконуються при старті і гальмують завантаження.

```powershell
# Завдання що запускаються при старті або вході
Get-ScheduledTask |
  Where-Object { $_.Triggers.CimClass -match "Boot|Logon" -and $_.State -eq "Ready" } |
  Select-Object TaskName, TaskPath |
  Format-Table -AutoSize
```

Перевір список — вимкни незнайомі або непотрібні завдання через `taskschd.msc`.

---

## Крок 7: Перевір цілісність системних файлів

Пошкоджені системні файли — причина повільного завантаження.

```cmd
DISM /Online /Cleanup-Image /RestoreHealth
sfc /scannow
```

Детальніше: [SFC і DISM — відновлення системи](/sfc-dism-vidnovlennya-systemnykh-fayliv)

---

## Крок 8: Аналіз завантаження через Windows Performance Recorder

Для глибокого аналізу — профілювання завантаження:

```cmd
rem Записати трасування завантаження
xbootmgr -trace boot -traceFlags BASE+CSWITCH+DRIVERS+POWER -resultPath C:\traces
```

Або встанови **Windows Performance Analyzer** (WPA) — частина Windows ADK — і відкрий файл trace.

---

## Що ще допомагає

**Дефрагментація** — тільки для HDD (SSD не дефрагментують):
`Параметри` → `Система` → `Сховище` → `Оптимізувати диски` → вибери HDD → **Оптимізувати**

**Збільшити RAM** — якщо менше 8 ГБ і Windows використовує файл підкачки при завантаженні.

**Оновити BIOS** — іноді нова версія BIOS пришвидшує POST (самоперевірку перед завантаженням Windows).

**Вимкнути непотрібні пристрої в BIOS** — зчитувач карт, COM-порти, мережеве завантаження (PXE Boot).

---

## Очікуваний результат після оптимізації

| Конфігурація | До | Після |
|---|---|---|
| HDD + 10 програм автозапуску | 90–120 сек | 60–75 сек |
| SSD + 10 програм автозапуску | 25–35 сек | 10–15 сек |
| SSD + мінімум автозапуску | 15–20 сек | 8–12 сек |

---

## Підсумок

Найбільший ефект по спадній: SSD замість HDD → вимкнути зайве з автозапуску → увімкнути швидкий запуск → SFC/DISM. Перші три кроки дають 80% результату і займають 10 хвилин.
