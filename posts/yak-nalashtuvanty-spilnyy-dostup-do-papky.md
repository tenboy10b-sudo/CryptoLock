Мережева папка (шаринг) дозволяє ділитись файлами між комп'ютерами в одній мережі без флешок і хмарних сервісів. Налаштовується за 5 хвилин.

---

## Частина 1: Відкрити папку для спільного доступу

### Крок 1: Увімкни мережеве виявлення

`Пуск` → `Параметри` → `Мережа і Інтернет` → `Додаткові параметри мережі` → `Додаткові параметри спільного доступу`

Увімкни для **Приватна мережа**:
- ✅ Мережеве виявлення
- ✅ Спільний доступ до файлів і принтерів

### Крок 2: Відкрий доступ до папки

Клікни правою кнопкою на папку → **Властивості** → вкладка **Спільний доступ** → **Спільний доступ...**

У вікні введи ім'я користувача або вибери:
- `Everyone` — доступ для всіх у мережі
- Конкретний обліковий запис — для окремого користувача

Встанови рівень дозволів:
- **Читання** — тільки переглядати файли
- **Читання і запис** — переглядати і змінювати

Натисни **Спільний доступ** → **Готово**.

### Крок 3: Налаштуй дозволи NTFS (за потреби)

Вкладка **Безпека** → **Редагувати** → додай користувача і встанови дозволи NTFS.

> Права NTFS і права шарингу діють одночасно — застосовуються більш обмежувальні.

Детальніше про NTFS: [Права доступу до файлів і папок](/prava-dostupu-fayly-papky)

## Частина 2: Підключитися до мережевої папки

### Спосіб 1: Через Провідник

`Win + E` → у адресному рядку введи:

```
\\ім'я-комп'ютера\назва-папки
```

або по IP:

```
\\192.168.1.100\Documents
```

Як дізнатись ім'я комп'ютера: `Win + R` → `sysdm.cpl` → вкладка **Ім'я комп'ютера**.

### Спосіб 2: Підключити як мережевий диск

Щоб папка завжди була видна як диск (наприклад Z:):

`Win + E` → **Цей комп'ютер** → вкладка **Комп'ютер** (або через меню `...`) → **Підключити мережевий диск**

- Виберіть літеру диска
- Введи шлях: `\\192.168.1.100\Documents`
- Постав ✅ **Підключати при вході в систему**

### Спосіб 3: Через CMD

```cmd
:: Підключити мережевий диск
net use Z: \\192.168.1.100\Documents

:: З авторизацією
net use Z: \\192.168.1.100\Documents /user:Username Password

:: Відключити
net use Z: /delete

:: Показати всі підключення
net use
```

## Типові проблеми

### "Немає доступу до мережевої папки"

1. Перевір що мережеве виявлення увімкнене на обох комп'ютерах
2. Перевір що обидва ПК в одній мережі (та сама Wi-Fi мережа або кабель до одного роутера)
3. Перевір брандмауер — він може блокувати SMB. Дозволь через: [Брандмауер Windows через GPO](/brandmauer-windows-cherez-gpo)
4. Спробуй підключитись по IP замість імені ПК

### Запитує пароль хоча встановив "Everyone"

Якщо на цільовому ПК захищений обліковий запис — Windows вимагає авторизацію. Введи логін і пароль того комп'ютера.

Або: `Параметри` → `Мережа` → `Додаткові параметри спільного доступу` → **Спільний доступ, захищений паролем** → вимкни (тільки для домашньої мережі!).

### Папка не видна в мережі

Відкрий порти SMB у брандмауері:

```cmd
netsh advfirewall firewall add rule name="SMB" dir=in action=allow protocol=TCP localport=445
```

### Повільна передача файлів по мережі

- По Wi-Fi норма: 5–10 МБ/с. По кабелю: 100–120 МБ/с (Fast Ethernet) або 1 ГБ/с (Gigabit).
- Перевір що обидва комп'ютери підключені до гігабітного роутера і кабелі Cat5e або Cat6.

## Для корпоративного середовища

У домені Active Directory спільний доступ до папок зручніше налаштовувати через DFS (Distributed File System) або через GPO.

Основи GPO: [Групова політика для користувачів домену](/gpo-obmezhennya-dlya-koristuvachiv-domenu)

## Підсумок

Шаринг папки: правою кнопкою → Властивості → Спільний доступ → додати користувача → вибрати права. Підключення: `\\IP\папка` у Провіднику або `net use` з CMD.
