Комп'ютер гальмує але незрозуміло що саме завантажене? Розберемо всі інструменти для моніторингу продуктивності Windows — від простих до детальних.

## Спосіб 1: Диспетчер завдань (швидкий огляд)

`Ctrl + Shift + Esc` → вкладка **Продуктивність**.

Тут бачиш в реальному часі:

| Показник | На що звертати увагу |
|---|---|
| **CPU** | Якщо постійно вище 80–90% — щось навантажує процесор |
| **Пам'ять** | Якщо використовується більше 85% — мало RAM |
| **Диск** | 100% завантаження диска = серйозне гальмування |
| **Мережа** | Показує швидкість завантаження/відправки |
| **GPU** | Завантаженість відеокарти |

Натисни **Відкрити монітор ресурсів** внизу — для детальнішої картини.

---

## Спосіб 2: Монітор ресурсів (детальний)

`Win + R` → `resmon` → Enter.

Тут 4 вкладки:

**CPU** — які процеси скільки використовують процесор. Шукай процес з найвищим %CPU.

**Пам'ять** — використання RAM по процесах. Колонка **Робочий набір** показує скільки RAM займає процес.

**Диск** — які файли зчитуються/записуються прямо зараз. Якщо диск 100% — побачиш який процес винен.

**Мережа** — які програми використовують інтернет і скільки трафіку.

---

## Спосіб 3: Монітор продуктивності (для аналізу)

`Win + R` → `perfmon` → Enter.

Більш складний інструмент але дозволяє:
- Записувати показники продуктивності за часом
- Будувати графіки навантаження
- Порівнювати показники

Для швидкого звіту: у лівій панелі **Звіти** → **Система** → **Перевірка системи** — Windows створить детальний звіт про стан ПК.

## Спосіб 4: Через PowerShell

**Завантаженість CPU:**
```powershell
Get-Counter '\Processor(_Total)\% Processor Time' -SampleInterval 1 -MaxSamples 5
```

**Використання RAM:**
```powershell
Get-Counter '\Memory\Available MBytes'
```

**Топ процесів по CPU:**
```powershell
Get-Process | Sort-Object CPU -Descending | Select-Object -First 10 Name, CPU, WorkingSet
```

**Завантаженість диска:**
```powershell
Get-Counter '\PhysicalDisk(_Total)\% Disk Time'
```

## Спосіб 5: Оцінка продуктивності Windows

В старих версіях Windows був індекс продуктивності. Зараз можна отримати схожі дані через CMD:

```cmd
winsat formal
```

Після завершення:
```cmd
winsat query
```

Результати збережуться в `C:\Windows\Performance\WinSAT\DataStore\`.

## Що означають показники

**CPU 100%** — процес завис або система перевантажена. Знайди в Диспетчері завдань який процес винен.

**Диск 100%** — найчастіше причина: Windows Search або Superfetch (SysMain). Можна вимкнути службу `SysMain` якщо є SSD.

**RAM понад 90%** — потрібно більше пам'яті або закрити зайві програми.

**Температура** — Диспетчер завдань не показує температуру, для цього використовуй HWiNFO64 або HWMonitor.
