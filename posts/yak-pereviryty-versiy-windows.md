Іноді потрібно знати точну версію Windows — для встановлення драйвера, вирішення проблеми або перевірки сумісності. Ось найшвидші способи.

---

## Найшвидший спосіб: winver

`Win + R` → `winver` → Enter

Відкривається вікно з назвою видання (Home / Pro / Enterprise) і номером збірки.

---

## Детальна інформація: Параметри

`Win + I` → **Система** → **Про систему** → прокрути вниз до **Специфікації Windows**:

- **Видання**: Windows 11 Home / Pro
- **Версія**: 23H2 (назва оновлення)
- **Встановлено**: дата встановлення
- **Збірка ОС**: 22631.xxxx (точний номер збірки)
- **Виробник**: якщо ноутбук або брендовий ПК

---

## Через CMD

```cmd
ver
```

Виводить щось на зразок: `Microsoft Windows [Version 10.0.22631.3880]`

Де:
- `10.0` — внутрішній номер (навіть Windows 11 показує 10.0)
- `22631` — номер збірки
- `3880` — номер патча

---

## Через PowerShell (найбільше деталей)

```powershell
Get-ComputerInfo | Select-Object WindowsProductName, WindowsVersion, OsBuildNumber, OsArchitecture
```

Або коротко:
```powershell
[System.Environment]::OSVersion
```

---

## Перевірити архітектуру (32 чи 64 біт)

```powershell
(Get-WmiObject Win32_OperatingSystem).OSArchitecture
```

Виводить `64-bit` або `32-bit`. Більшість сучасних систем — 64-біт.

---

## Таблиця версій Windows 11

| Версія | Збірка | Дата |
|--------|--------|------|
| 24H2 | 26100 | 2024 |
| 23H2 | 22631 | 2023 |
| 22H2 | 22621 | 2022 |
| 21H2 | 22000 | 2021 |

Якщо твоя збірка нижча за 22631 — система потребує оновлення.

---

## Резюме

Для швидкої перевірки — `winver`. Для точного номера збірки — **Параметри → Система → Про систему**. Для скриптів і автоматизації — PowerShell `Get-ComputerInfo`.
