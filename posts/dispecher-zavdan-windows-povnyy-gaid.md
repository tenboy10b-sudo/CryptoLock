Більшість людей відкривають Диспетчер завдань тільки щоб завершити завислу програму. Але він вміє набагато більше — ось що ти пропускаєш.

---

## Як відкрити

- `Ctrl + Shift + Esc` — найшвидший спосіб
- `Ctrl + Alt + Delete` → Диспетчер завдань
- Правою кнопкою на панелі завдань → Диспетчер завдань
- `Win + R` → `taskmgr`

---

## Вкладка Процеси

Стандартне завершення процесу — правою кнопкою → **Завершити задачу**.

**Але є ще:**

```
Правою кнопкою на процес:
→ Перейти до деталей — відкриє вкладку Деталі для цього процесу
→ Відкрити розташування файлу — покаже де знаходиться .exe
→ Пошук в Інтернеті — гуглить назву процесу (зручно для невідомих)
→ Властивості — деталі файлу, підпис
```

**Стовпці які варто додати** (правою кнопкою на заголовок):
- **GPU** — хто навантажує відеокарту
- **Енергоспоживання** — хто садить батарею
- **PID** — ідентифікатор процесу

---

## Вкладка Продуктивність

**CPU** — завантаження, частота, температура (тільки Windows 11), кількість ядер/потоків, час роботи системи.

**Пам'ять** — зайнято, доступно, швидкість RAM, кількість слотів. Корисно: якщо "Слоти, що використовуються: 2 з 2" і пам'яті мало — апгрейд неможливий без заміни планок.

**Диск** — активний час, швидкість читання/запису. Якщо активний час постійно 100% — диск є вузьким місцем.

**Мережа** — поточна швидкість передачі по кожному адаптеру.

**GPU** — завантаження, температура, VRAM.

Знизу є посилання **Відкрити монітор ресурсів** — детальніший аналіз.

## Вкладка Користувачі

Показує всіх підключених користувачів і їх ресурси. Корисно на RDP серверах.

```
Правою на сесію:
→ Відключити — від'єднати сесію (сесія залишається активною)
→ Вийти — завершити сесію
→ Надіслати повідомлення — відправити текстове повідомлення користувачу
```

---

## Вкладка Деталі

Детальніша версія вкладки Процеси. Тут можна:

```
Правою на процес:
→ Встановити пріоритет — підвищити або знизити пріоритет CPU
→ Встановити подібність — закріпити процес за конкретними ядрами CPU
→ Проаналізувати ланцюжок очікування — показати чому процес завис
→ Аварійне завершення процесу — примусово без збереження
```

**Встановити пріоритет** для ігри або рендерингу:
```
Правою → Встановити пріоритет → Вище середнього або Високий
```

## Вкладка Служби

Швидкий перегляд і управління службами без відкриття services.msc.

Правою → **Запустити/Зупинити** або **Відкрити служби** для повного управління.

## Команди taskkill і tasklist

```cmd
rem Список всіх процесів
tasklist

rem Знайти конкретний процес
tasklist | findstr chrome

rem Завершити процес за іменем
taskkill /f /im chrome.exe

rem Завершити за PID
taskkill /f /pid 1234

rem Завершити всі процеси певного користувача
taskkill /f /fi "USERNAME eq john"
```

## PowerShell аналог

```powershell
# Список процесів відсортований по CPU
Get-Process | Sort-Object CPU -Descending | Select-Object -First 10 Name, CPU, WorkingSet

# Завершити процес
Stop-Process -Name "chrome" -Force

# Процеси що використовують більше 500 МБ RAM
Get-Process | Where-Object { $_.WorkingSet -gt 500MB } |
  Sort-Object WorkingSet -Descending |
  Select-Object Name, @{N="RAM MB";E={[math]::Round($_.WorkingSet/1MB)}}
```

## Діагностика гальмувань через Диспетчер

**Сценарій: ПК гальмує, не знаю чому**

1. Відкрий → вкладка **Продуктивність**
2. Перевір CPU — якщо > 80% постійно → вкладка **Процеси** → сортуй по CPU
3. Перевір Пам'ять — якщо > 90% → мало RAM або витік пам'яті
4. Перевір Диск — якщо Активний час 100% → HDD перевантажений
5. Перевір GPU — якщо несподівано 100% → фонова задача використовує GPU

## Підсумок

`Ctrl + Shift + Esc` → **Продуктивність** → дивишся де вузьке місце. **Деталі** → правою → **Проаналізувати ланцюжок очікування** — знаходиш чому завис процес. Колонки CPU, GPU і Енергоспоживання у вкладці Процеси — додай і завжди бачиш хто що їсть.
