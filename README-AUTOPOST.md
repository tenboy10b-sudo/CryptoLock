# Автопостинг статей

## Як це працює

Стаття з'являється на сайті тільки коли настає її `publishDate`.

Приклад майбутньої статті:
```markdown
---
title: "Назва статті"
date: "2026-05-01"
publishDate: "2026-05-01"
description: "Опис..."
tags: ["windows", "безпека"]
readTime: 5
---
Текст статті...
```

- `date` — дата для відображення читачеві
- `publishDate` — дата коли стаття стає видимою на сайті

Якщо `publishDate` не вказано — стаття публікується одразу.

## Workflow для запланованих статей

1. Напиши статтю з `publishDate` у майбутньому
2. Зроби `git push` — стаття потрапляє в репо але НЕ відображається
3. Vercel щодня робить автоматичний redeploy (налаштуй нижче)
4. Коли настає дата — стаття автоматично з'являється

## Налаштування щоденного redeploy на Vercel

Щоб статті з'являлись точно в дату — потрібно щоб Vercel перебудовував сайт щодня.

### Через Vercel Deploy Hooks:

1. Vercel → твій проект → Settings → Git → Deploy Hooks
2. Натисни "Create Hook" → назви "Daily Rebuild" → Branch: main → Copy URL
3. Зайди на cron-job.org → Create cronjob
4. URL: вставте скопійований webhook
5. Schedule: Every day at 00:05
6. Save

Тепер о 00:05 щодня Vercel автоматично перебудує сайт і нові статті з'являться.

## Швидка публікація без чекання

Якщо хочеш опублікувати статтю одразу — просто не вказуй `publishDate` або постав сьогоднішню дату.
